Workflow

10 ขั้นตอนการทำ Machine Learning Pipeline

ขั้นสำรวจและจัดเตรียม (Step 1-4)
  1. Setup & Imports: ติดตั้งและดึงคลังไลบรารี
  2. Load Dataset: นำข้อมูลเข้าสู่ DataFrame
  3. Quick Look: สำรวจค่านูลและสถิติเบื้องต้น
  4. Train / Test Split: สุ่มแบ่งกลุ่มควบคุมวัดผล
ขั้นฝึกฝนและทดสอบ (Step 5-10)
  1. Preprocessing Pipeline: จัดการค่าว่างและปรับสเกล
  2. Compare Models: คัดเลือกโมเดลที่ดีที่สุดด้วย CV
  3. Hyperparameter Tuning: จูนค่าเหมาะสมด้วย Grid Search
  4. Evaluate on Test Set: ประเมินผลลัพธ์ด่านสุดท้าย
  5. Feature Importance: ตรวจสอบอิทธิพลของแต่ละปัจจัย
  6. Inference & Production: เซฟโมเดลและใช้งานทำนาย

ทำไมไม่ต้องใช้ stratify? เนื่องจากข้อมูลเป้าหมายคือน้ำหนักในหน่วยกรัม (Continuous Variable) จึงไม่มีกลุ่มคลาสที่แน่นอนในการสุ่มเกลี่ยสัดส่วน

Step 5

การจัดท่อเตรียมข้อมูล (Preprocessing Pipeline)

การจัดสรรชุดคอลัมน์ตามหมวดหมู่เพื่อส่งผ่านท่อกรองข้อมูลย่อยที่แตกต่างกันด้วย [ColumnTransformer]:

# จัดท่อสำหรับข้อมูลตัวเลข (Numeric Transformer)
numeric_transformer = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="median")),  # เติมค่าว่างด้วยค่ากลาง (มัธยฐาน)
    ("scaler", StandardScaler())                    # ปรับขนาดข้อมูลให้มีความเบี่ยงเบนมาตรฐานเป็น 1
])

# จัดท่อสำหรับข้อมูลหมวดหมู่ (Categorical Transformer)
categorical_transformer = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="most_frequent")), # เติมค่าว่างด้วยค่าฐานนิยม
    ("onehot", OneHotEncoder(handle_unknown="ignore", sparse_output=False)) # แปลงเป็นเลข 0 และ 1
])

# รวบรวมท่อทั้งสองเข้าด้วยกันตามกลุ่มคอลัมน์ฟีเจอร์ที่ระบุ
preprocessor = ColumnTransformer(transformers=[
    ("num", numeric_transformer, numeric_features),
    ("cat", categorical_transformer, categorical_features)
])
Lab 1 Insight

การตัดสินใจออกแบบ: กรณีศึกษาตัวแปร `pclass`

ตัวแปร `pclass` (ระดับชั้นตั๋วผู้โดยสาร 1, 2, 3) ใน Titanic มีลักษณะเป็นข้อมูลเชิงอันดับ (Ordinal Data)

แนวทางแบบ Nominal (One-Hot)

หากแปลง `pclass` เป็น One-Hot จะแตกออกเป็น 3 คอลัมน์อิสระกันได้แก่ `pclass_1`, `pclass_2`, `pclass_3` และมีค่า 0 หรือ 1 ขาดความต่อเนื่องเชิงมิติ

ผลลัพธ์: สูญเสียระดับความสัมพันธ์ของชั้นตั๋ว (ตัวอย่างเช่น ตั๋วชั้น 1 มีสิทธิรอดสูงกว่าชั้น 3)

แนวทางแบบ Ordinal (Numeric Scaling)

ในเวอร์ชัน v2 โค้ดย้าย `pclass` มาประมวลผลในท่อนิวเมอริก เพื่อผ่านการทำ `StandardScaler` เพื่อปรับขนาดค่าช่วงความกว้าง

ผลลัพธ์: รักษามิติเชิงอันดับและทิศทางของข้อมูล (1 < 2 < 3) ส่งผลให้แบบจำลองเชิงเส้นเรียนรู้ค่าน้ำหนักได้แม่นยำยิ่งขึ้น

Step 7

การปรับจูนพารามิเตอร์ของโมเดลตัวที่เก่งสุด (GridSearchCV)

ในเวอร์ชัน v2 โค้ดจะดึงคลาสโมเดลและเลือกชุดตารางจูนพารามิเตอร์ให้ตรงกับตัวที่ชนะใน Step 6 แบบอัตโนมัติ (Dynamic Tuning):

# กำหนดพารามิเตอร์แยกตามกลุ่มโมเดลหลัก
param_grids = {
    "Logistic Regression": {"model__C": [0.01, 0.1, 1.0, 10.0]},
    "Random Forest":       {"model__n_estimators": [50, 100, 200], "model__max_depth": [None, 5, 10]}
}

# ดึงข้อมูลมาจับคู่เพื่อส่งต่อเข้าท่อส่ง Grid
best_model_class = type(models[best_model_name])

final_pipe = Pipeline(steps=[
    ("preprocessor", preprocessor),
    ("model", best_model_class(random_state=42))

grid_search = GridSearchCV(final_pipe, selected_grid, cv=5, scoring="accuracy")

Lab 1 (Step 8)

ประเมินผลลัพธ์สุดท้าย: สถิติ Classification

การวิเคราะห์แบบจำลอง Titanic ด้วย Confusion Matrix และรายงานผลสถิติ:

📊 ตารางสถิติผู้โดยสารในกลุ่มทดสอบ (Test Set)

- True Negative (TN): ทำนายเสียชีวิต และเสียชีวิตจริง = 103 คน
- False Positive (FP): ทำนายรอดชีวิต แต่ผลจริงคือเสียชีวิต = 7 คน
- False Negative (FN): ทำนายเสียชีวิต แต่ผลจริงคือรอดชีวิต = 29 คน
- True Positive (TP): ทำนายรอดชีวิต และรอดชีวิตจริง = 40 คน

🧩 ความสำคัญของ Precision และ Recall

- Precision (คลาสคนรอดชีวิต = 85%): ในกลุ่มคนที่โมเดลพยากรณ์ว่ารอดชีวิต โมเดลคาดเดาได้ตรงกับความจริง 85%
- Recall (คลาสคนรอดชีวิต = 58%): จากผู้รอดชีวิตทั้งหมดในเรือ โมเดลสแกนตามตัวเจอเพียง 58% เท่านั้น

Step 9

การวิเคราะห์ปัจจัยและถอดรหัสโมเดลอย่างปลอดภัย

ในเวอร์ชัน v2 โค้ดจะใช้ฟังก์ชันตรวจสอบสถาปัตยกรรมโมเดลอย่างปลอดภัยด้วยเงื่อนไข `hasattr()`:

# 1) ตรวจสอบความเข้ากันได้ของสถาปัตยกรรมเพื่อป้องกันโมเดลแครช
if hasattr(model, "feature_importances_"):
    importances = model.feature_importances_  # สำหรับโมเดลประเภท Tree (RF, DT)
elif hasattr(model, "coef_"):
    importances = np.abs(model.coef_).flatten()  # สำหรับโมเดลเชิงเส้น (Logistic/Linear)
else:
    importances = None

Step 9.5 & 10

การส่งผ่านระบบและการนำโมเดลไปใช้งานจริง (Production)

ในการผลิตซอฟต์แวร์จริง เราจำเป็นต้องสร้างระบบที่มีความเสถียรและทนทาน:

1. Model Serialization (joblib)

การฝึกสอนโมเดล (Training) ใช้พลังงานสูงและเวลามาก ในงานทำนายหลังบ้าน เราจึงบันทึกผลการจูนเก็บเป็นไฟล์รูปเล่ม `.joblib` แล้วเปิดคำสั่ง `joblib.load()` เพื่อนำมาพยากรณ์ผลลัพธ์ใหม่ได้ทันทีโดยไม่ต้องรันเทรนซ้ำ

2. Input Validation

สร้างฟังก์ชันด่านคัดกรองข้อมูลขาเข้า เช่น `validate_passenger_input()` และ `validate_penguin_input()` คอยแจ้งเตือนและสกรีนตัวแปรแปลกปลอมหรือค่าว่างก่อนส่งเข้าสู่แบบจำลองคณิตศาสตร์ ป้องกันระบบหยุดทำงานกลางคัน

  • GridSearchCV: ค้นหารุ่นพารามิเตอร์ดีที่สุดอัตโนมัติ
  • Evaluation: เข้าใจความต่างของค่าวัด Classification และ Regression
